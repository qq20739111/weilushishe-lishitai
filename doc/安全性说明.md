# 围炉诗社·理事台 安全性说明

本文档说明"围炉诗社·理事台"系统在安全方面的设计和保护措施，帮助用户了解系统如何保障数据安全和使用安全。

---

## 一、总体说明

本系统运行在本地局域网环境中，不直接连接互联网，从源头上减少了来自外部的安全威胁。系统在设计上采用了**权限分级、密码加密存储、登录日志记录**等多项安全措施，确保数据和操作的安全可控。

---

## 二、网络安全

### 1. WiFi 连接保护
- 系统的 WiFi 热点使用 **WPA2 加密**（目前主流的无线安全标准），连接需要输入正确的密码。
- WiFi 密码要求至少 8 位，符合安全规范。

### 2. 局域网运行
- 系统仅在本地网络中运行，外部网络无法直接访问，大大降低了被攻击的风险。
- 如果 WiFi 连接断开，系统会自动尝试重新连接，保障服务持续可用。

### 3. 异常请求拦截
- 系统会自动拦截异常大小的网络请求，防止恶意数据占用系统资源。

### 4. 浏览器安全防护
- 系统自动设置了浏览器安全策略，防止页面被其他网站恶意嵌套，防止数据被篡改。
- 敏感接口的响应数据不会被浏览器缓存，避免信息残留。

### 5. 系统自我恢复
- 系统内置了"看门狗"保护机制：如果系统出现异常无响应，会在数分钟内自动重启恢复，避免长时间停机。

---

## 三、用户使用安全

### 1. 密码保护
- 用户密码经过**加密处理后存储**，即使数据文件被直接查看，也无法获知原始密码。
- 密码要求至少 6 位，且必须包含至少两种不同类型的字符（如数字+字母），防止过于简单的密码。

### 2. 登录凭证管理
- 登录成功后，系统会发放一个有时效的"通行证"（登录凭证），有效期为 30 天，过期后需重新登录。
- 系统重启后，所有登录凭证自动失效，需要重新登录，进一步保障安全。

### 3. 权限分级管理
系统将用户分为五个权限等级，不同等级能执行的操作不同：

| 角色 | 权限范围 |
|------|---------|
| 超级管理员 | 拥有所有操作权限 |
| 管理员 | 管理系统设置、成员等 |
| 理事 | 管理诗歌、活动、查看日志等 |
| 财务 | 管理财务相关功能 |
| 社员 | 基础使用权限 |

- 高权限角色不能通过接口随意创建，存在严格的分配限制。
- 普通用户无法修改他人信息或执行管理操作。

### 4. 敏感操作二次确认
- 删除数据、修改重要信息等操作，系统会弹出确认提示，防止误操作。

### 5. 登录日志
- 系统记录每一次登录行为（包括成功和失败），记录内容包括登录时间、来源地址等。
- 日志中的手机号做了隐藏处理（如 `138****2018`），保护隐私。
- 理事及以上权限的管理人员可以查看登录日志，及时发现异常登录行为。

### 6. 维护模式
- 系统支持"维护模式"，开启后普通用户无法登录和访问，仅管理员可操作，适用于数据维护或系统升级期间。

### 7. 游客访问控制
- 系统可配置是否允许未登录的用户浏览公开内容（如诗歌、活动信息）。
- 即使允许游客浏览，写入和管理操作仍需登录后才能执行。

---

## 四、使用建议

1. **设置强密码** - 请为 WiFi 和管理员账号设置足够复杂的密码，避免使用简单的数字或常见词组。
2. **定期更换密码** - 建议定期修改管理员账号密码，降低密码泄露风险。
3. **关注登录日志** - 定期查看登录日志，如发现不认识的登录记录，请及时修改密码。
4. **保持局域网使用** - 本系统设计为局域网内使用，不建议将设备直接暴露到互联网。
5. **妥善保管设备** - 系统运行在物理设备上，请将设备放置在安全的位置，避免被未授权人员接触。
