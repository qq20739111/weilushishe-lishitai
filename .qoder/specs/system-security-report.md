# 安全性报告编写计划

## 目标
为"围炉诗社·理事台"系统编写一份面向非技术用户的安全性说明报告，涵盖网络安全和用户使用安全两大方面，语言通俗易懂，结构精简。

## 报告存放位置
`doc/安全性说明.md`

## 报告大纲

### 一、概述
简要说明系统的安全设计理念（局域网运行、权限分级、数据保护）。

### 二、网络安全
1. **WiFi 连接保护** - 系统使用加密的 WiFi 连接（WPA2），热点有密码保护
2. **局域网运行** - 系统运行在本地网络中，不暴露到公网，降低外部攻击风险
3. **请求防护** - 系统对异常大小的请求进行拦截，防止恶意攻击
4. **安全响应头** - 防止页面被嵌套、防止内容类型伪造等浏览器层面防护
5. **系统自我保护** - 看门狗机制，系统异常时自动重启恢复

### 三、用户使用安全
1. **账号密码安全** - 密码加密存储，不以明文保存；密码强度要求（至少6位，至少两种字符类型）
2. **登录凭证（Token）** - 自动过期机制（30天），过期后需重新登录；服务器重启后凭证自动失效
3. **权限分级管理** - 五级权限（超级管理员 > 管理员 > 理事 > 财务 > 社员），不同操作需要不同权限
4. **敏感操作确认** - 删除、重要修改等操作需二次确认
5. **登录日志** - 记录每次登录（成功/失败），手机号脱敏显示，可追溯异常登录
6. **维护模式** - 系统维护期间仅管理员可登录，保护数据安全
7. **游客访问控制** - 可配置是否允许未登录用户查看公开内容

### 四、注意事项与建议
1. 系统运行在本地网络，请确保 WiFi 密码足够强
2. 定期修改管理员密码
3. 不建议将系统直接暴露到互联网
4. 关注登录日志，发现异常及时处理
